# 第一章 快速入门

本章帮助您在 5 分钟内完成从登录到开单的核心流程。

## 1.1 访问系统

在浏览器地址栏输入系统地址：

- 本地开发环境：`http://localhost`
- 部署环境：请向系统管理员获取实际访问地址

> 提示：推荐使用 Chrome、Edge 等现代浏览器访问，以获得最佳体验。

## 1.2 登录系统

1. 打开系统地址后，浏览器将自动跳转到登录页面。
2. 在登录页面输入用户名和密码。
3. 点击「登录」按钮。

系统预置了默认管理员账号：

| 项目 | 值 |
|------|------|
| 用户名 | `admin` |
| 密码 | `admin123` |

> 警告：首次登录后，请尽快修改默认密码，以确保账户安全。

## 1.3 创建一个商品

1. 在左侧菜单中点击「商品管理」。
2. 点击页面右上角的「新增商品」按钮。
3. 填写以下信息：
   - **商品名称**（必填）：例如"纯棉T恤"
   - **商品图片**（必填）：点击「选择图片」上传商品照片
   - **成本价**（必填）：例如 45.00
   - **销售价**（必填）：例如 99.00
4. 点击「创建商品」完成添加。

> 提示：SKU、库存数量、排序权重等高级选项默认由系统自动处理。如需自定义，可点击「展开高级设置」。

## 1.4 创建一个客户

1. 在左侧菜单中点击「客户管理」。
2. 点击「新增客户」。
3. 至少填写**客户名称**（必填），例如"张三公司"。
4. 根据需要填写联系人、电话、来源、等级等信息。
5. 点击「创建客户」完成添加。

## 1.5 创建一个订单

1. 在左侧菜单中点击「订单管理」。
2. 点击「新建订单」。
3. 在客户栏搜索并选择客户（也可以不选，表示散客消费）。
4. 选择收款方式（现金/转账/微信/支付宝/其他）。
5. 点击「添加商品」，从商品列表中选择要销售的商品。
6. 调整每项商品的**数量**和**成交单价**（默认使用商品销售价）。
7. 确认合计金额无误后，点击「确认销售」。
8. 系统自动确认订单并扣减库存；如果选择了收款方式，系统会自动创建收款记录。

> 至此，您已完成从商品录入到销售开单的完整流程。接下来请阅读各章节了解系统的全部功能。

# 第二章 登录与账户安全

## 2.1 登录页面

系统登录页面包含以下元素：

- **系统标题**：页面顶部显示"销售管理系统"
- **用户名输入框**：带有用户图标的输入框，提示文字为"用户名"
- **密码输入框**：带有锁图标的密码输入框，提示文字为"密码"，输入内容自动隐藏
- **登录按钮**：点击提交登录信息

## 2.2 登录步骤

1. 在用户名输入框中输入您的用户名。
2. 在密码输入框中输入您的密码。
3. 点击「登录」按钮或按 Enter 键提交。

登录成功后，系统将跳转至首页仪表盘。

## 2.3 安全机制

| 安全策略 | 说明 |
|----------|------|
| 登录锁定 | 连续 5 次输入错误密码后，账户将被锁定 15 分钟 |
| Token 过期 | 登录令牌（Token）有效期为 30 分钟 |
| 自动续期 | 在 Token 即将过期时，系统会自动刷新，无需重复登录 |
| 自动登出 | Token 失效后，系统将自动跳转到登录页面 |

> 提示：如果您长时间未操作页面后被自动登出，只需重新输入用户名和密码登录即可。

# 第三章 首页仪表盘

首页仪表盘是登录后看到的第一个页面，提供关键业务指标的实时概览。

## 3.1 页面布局

页面顶部左侧显示标题"首页看板"，右侧显示**时间周期选择器**，可选择以下时间段查看数据：

| 选项 | 含义 |
|------|------|
| 今日 | 当天 0:00 至当前时刻 |
| 近 7 天 | 最近 7 个自然日 |
| 近 30 天 | 最近 30 个自然日（默认） |
| 本月 | 当月 1 日至当前时刻 |
| 上月 | 上一个完整自然月 |

## 3.2 统计卡片

页面第一行展示 3 至 4 张统计卡片：

| 卡片 | 说明 |
|------|------|
| 销售总额 | 所选周期内的全部订单金额合计 |
| 订单数 | 所选周期内的订单数量 |
| 毛利 | 销售总额减去总成本的金额（绿色为正，红色为负） |
| 毛利率 | 毛利占销售总额的百分比（需有相应权限才可见） |

> 提示：毛利率卡片仅对拥有"报表利润查看"权限的用户可见。

## 3.3 销售趋势图

页面中部左侧展示**销售趋势**条形图：

- 每根柱子代表一天的销售金额。
- 蓝色柱子表示有销售额，灰色柱子表示销售额为零。
- 图表上方显示"期间总额"和"峰值日"信息。
- 鼠标悬停在柱子上可查看具体日期、金额和订单数。
- 最长展示 30 天的数据。

## 3.4 库存预警

页面中部右侧展示**库存预警**列表：

- 显示库存数量低于或等于预警阈值（默认为 10）的商品。
- 库存为 0 的商品标记为红色标签。
- 库存低于阈值但大于 0 的商品标记为橙色标签。
- 列表包含 SKU、商品名称、当前库存和销售价。

## 3.5 商品销售排行

页面底部展示**商品销售排行**表格：

| 列 | 说明 |
|----|------|
| 排名 | 按销售额从高到低排序 |
| SKU | 商品编码 |
| 商品名称 | 商品的名称 |
| 销量 | 销售出的总数量 |
| 销售额 | 销售金额合计 |
| 毛利 | 销售额减去成本（绿色为正，红色为负） |

# 第四章 商品管理

商品管理用于维护系统中的全部商品信息，包括添加、编辑、导入、导出和停用商品。

## 4.1 商品列表

进入「商品管理」页面后，您可以看到商品列表，包含以下信息：

| 列 | 说明 |
|----|------|
| 图片 | 商品主图缩略图（50x50），无图片时显示"--" |
| SKU | 商品编码 |
| 商品名称 | 商品的名称 |
| 分类 | 商品所属分类 |
| 销售价 | 商品的销售价格 |
| 成本价 | 商品的成本价格（需有相应权限） |
| 利润 | 单件商品利润（需有相应权限） |
| 毛利率 | 利润占销售价的比例（需有相应权限） |
| 库存 | 当前库存数量 |
| 状态 | 上架（绿色）/ 下架（橙色）/ 停用（红色） |
| 操作 | 编辑、停用、删除 |

> 提示：成本价、利润、毛利率三列仅对拥有"查看成本价"权限的用户显示。

### 搜索与筛选

- **搜索**：在页面左上角的搜索框中输入关键词，可按商品名称模糊搜索。
- **状态筛选**：点击搜索框旁边的下拉菜单，可按"上架""下架""停用"三种状态筛选商品。
- 两种条件可以同时使用，实现精确查找。

### 分页

表格底部提供分页控件，默认每页显示 20 条，可切换为 10、50 或 100 条。右侧显示总记录数。

## 4.2 新增商品

1. 在商品列表页点击右上角的「新增商品」按钮。
2. 系统进入新增商品表单页面。
3. 填写商品信息：

#### 基础信息（必填）

| 字段 | 说明 | 规则 |
|------|------|------|
| 商品名称 | 商品的名称 | 必填，最多 100 个字符 |
| 商品图片 | 商品的主图 | 必填，支持 JPG/PNG/WebP 格式，最大 5MB |
| 成本价 | 商品的采购成本 | 必填，不能为负数，精确到分 |
| 销售价 | 商品的对外售价 | 必填，不能为负数，精确到分 |

#### 高级设置（选填）

点击「展开高级设置」可显示以下字段：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| SKU | 商品编码 | 留空时系统自动生成，格式为 `SPU-YYYYMMDD-NNNN` |
| 库存数量 | 初始库存 | 0 |
| 排序权重 | 列表排序依据，数值越大越靠前 | 0 |
| 状态 | 上架/下架/停用 | 上架 |
| 备注 | 补充说明文字 | 空 |

4. 点击「创建商品」完成添加。系统跳转回商品列表。

> 提示：新建商品的 SKU 建议留空，由系统自动生成以保证唯一性。编辑已有商品时，SKU 字段不可修改。

### 上传图片

1. 在商品图片区域点击「选择图片」按钮。
2. 在弹出的文件选择窗口中，选择一张商品图片。
3. 系统自动上传图片并显示预览。
4. 如需更换图片，点击「替换图片」按钮重新上传。

支持的图片格式和限制：

| 项目 | 要求 |
|------|------|
| 格式 | JPG、JPEG、PNG、WebP |
| 大小 | 最大 5MB |

## 4.3 编辑商品

1. 在商品列表中找到要编辑的商品，点击该行的「编辑」按钮。
2. 系统进入编辑商品表单页面，自动加载当前商品信息。
3. 修改需要更新的字段。
4. 点击「保存修改」完成编辑。

> 注意：编辑商品时 SKU 字段为不可修改状态（灰色），商品名称、成本价、销售价等字段均可修改。修改商品价格不会影响已创建的历史订单，历史订单保存的是下单时的价格快照。

## 4.4 停用商品

1. 在商品列表中找到状态为"上架"的商品。
2. 点击该行的「停用」按钮。
3. 系统将商品状态更改为"停用"。

> 说明：停用的商品将无法在创建订单时被选中，但已有订单中的该商品数据不受影响。

## 4.5 删除商品

1. 在商品列表中找到要删除的商品。
2. 点击该行的「删除」按钮。
3. 系统弹出确认对话框："确定删除该商品？"
4. 点击确定完成删除。

> 警告：已被订单引用的商品不建议删除。建议使用"停用"功能代替删除。

## 4.6 导入商品

批量导入商品的具体操作请参考 [第十三章 CSV 导入导出指南](#第十三章-csv-导入导出指南)。

## 4.7 导出商品

1. 在商品列表页点击右上角的「导出」按钮。
2. 系统将当前搜索/筛选条件下的商品数据导出为 CSV 文件。
3. 浏览器自动下载文件。

> 提示：导出范围受当前的搜索关键词和状态筛选条件影响。如果需要导出全部商品，请先清空搜索条件和状态筛选。

# 第五章 库存流水

库存流水页面记录系统中所有库存变动的详细历史。

## 5.1 页面布局

页面顶部左侧显示标题"库存流水"，右侧显示总记录数。

## 5.2 流水记录说明

每条库存变动记录包含以下信息：

| 列 | 说明 |
|----|------|
| 变动类型 | 手动调整（蓝色）/ 订单扣减（橙色）/ 取消退货（绿色） |
| 变动前 | 操作前的库存数量 |
| 变动量 | 变动的数量（绿色正数表示入库，红色负数表示出库） |
| 变动后 | 操作后的库存数量 |
| 关联类型 | 关联的业务对象类型，如"销售订单" |
| 备注 | 变动原因说明 |
| 时间 | 操作发生的精确时间 |

## 5.3 查看库存流水

1. 在左侧菜单中点击「库存流水」。
2. 系统默认展示最近的库存变动记录。
3. 使用表格底部的分页控件浏览更多记录。

> 说明：库存流水为系统自动记录，不支持手动添加或修改。以下操作会自动产生库存流水记录：
> - 创建订单并确认时：产生"订单扣减"记录
> - 取消已确认订单时：产生"取消退货"记录
> - 手动调整库存时：产生"手动调整"记录

# 第六章 客户管理

客户管理用于维护客户资料、跟踪客户状态并查看客户关联订单。

## 6.1 客户列表

进入「客户管理」页面后，您可以看到客户列表，包含以下信息：

| 列 | 说明 |
|----|------|
| 客户名称 | 点击可跳转到客户详情页 |
| 联系人 | 客户的主要联系人姓名 |
| 电话 | 联系电话 |
| 来源 | 转介绍/线上/线下/广告/其他 |
| 等级 | VIP（金色）/ 重要（蓝色）/ 普通（灰色）/ 潜在（绿色） |
| 归属销售 | 负责该客户的销售人员 |
| 跟进状态 | 新客户/跟进中/已成交/已流失 |
| 操作 | 编辑、删除 |

### 搜索与筛选

- **搜索**：在搜索框中输入关键词，可按客户名称或电话号码模糊搜索。
- **来源筛选**：通过下拉菜单按客户来源筛选（转介绍/线上/线下/广告/其他）。

### 分页

表格底部提供分页控件，默认每页 20 条，可切换为 10、50 或 100 条。

## 6.2 新增客户

1. 在客户列表页点击右上角的「新增客户」按钮。
2. 填写客户信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 客户名称 | 必填 | 客户或公司的名称，最多 100 个字符 |
| 联系人 | 选填 | 主要联系人姓名 |
| 电话 | 选填 | 联系电话，需为 11 位手机号格式（如 13800138000），且在系统中唯一 |
| 邮箱 | 选填 | 邮箱地址，需符合邮箱格式 |
| 来源 | 选填 | 转介绍/线上/线下/广告/其他，默认"其他" |
| 等级 | 选填 | VIP/重要/普通/潜在，默认"普通" |
| 跟进状态 | 选填 | 新客户/跟进中/已成交/已流失，默认"新客户" |
| 备注 | 选填 | 补充说明，最多 500 个字符 |

3. 点击「创建客户」完成添加。

> 提示：电话号码在系统中必须唯一。如果输入已存在的电话号码，系统会提示错误。

## 6.3 编辑客户

1. 在客户列表中找到要编辑的客户，点击该行的「编辑」按钮。
2. 修改需要更新的字段。
3. 点击「保存修改」完成编辑。

## 6.4 客户详情

1. 在客户列表中点击客户名称（蓝色链接），进入客户详情页。
2. 详情页上半部分展示客户基本信息：

| 信息项 | 说明 |
|--------|------|
| 联系人 | 主要联系人 |
| 电话 | 联系电话 |
| 邮箱 | 邮箱地址 |
| 来源 | 客户来源渠道 |
| 等级 | 客户等级标签 |
| 归属销售 | 负责的销售人员 |
| 跟进状态 | 当前的跟进状态 |
| 创建时间 | 客户创建的时间 |
| 更新时间 | 最近一次修改的时间 |
| 备注 | 补充说明 |

3. 详情页下半部分展示**关联订单**列表，包含该客户的全部历史订单：

| 列 | 说明 |
|----|------|
| 订单号 | 点击可跳转到订单详情 |
| 状态 | 草稿/已确认/已取消/部分收款/已完成 |
| 订单金额 | 订单总金额 |
| 创建时间 | 订单创建时间 |

4. 详情页右上角提供「编辑」和「删除」操作按钮。

## 6.5 删除客户

1. 在客户列表或客户详情页中点击「删除」按钮。
2. 系统弹出确认对话框："确定删除该客户？删除后不可恢复。"
3. 点击确定完成删除。

> 警告：已被订单关联的客户不建议删除。如需限制该客户的业务，可考虑修改客户等级或备注来标记。

## 6.6 导入客户

批量导入客户的具体操作请参考 [第十三章 CSV 导入导出指南](#第十三章-csv-导入导出指南)。

## 6.7 导出客户

## 7.2 创建订单

1. 在订单列表页点击右上角的「新建订单」按钮。
2. 系统进入创建订单页面。

### 选择客户（可选）

- 在「客户（可选）」下拉框中输入客户名称或电话进行搜索。
- 选中目标客户即可关联订单。
- 如果不选择客户，该订单将标记为"散客"订单。

> 提示：客户字段是可选的。对于未记录在系统中的散客消费，可以直接跳过此项。

### 选择收款方式

在「收款方式」下拉框中选择一种收款方式：

| 选项 | 含义 |
|------|------|
| 现金 | 现金收款 |
| 银行转账 | 银行转账收款 |
| 微信 | 微信支付 |
| 支付宝 | 支付宝支付 |
| 其他 | 其他收款方式 |

> 说明：如果选择了收款方式，系统在确认销售时会自动创建一条收款记录。如果不选择收款方式，则仅创建订单，不自动创建收款记录。

### 添加商品

1. 点击「添加商品」按钮。
2. 系统在页面下方展开商品选择面板。
3. 在搜索框中输入商品名称进行搜索（仅显示"上架"状态的商品）。
4. 找到目标商品后，点击该行的「选择」按钮。
5. 商品添加到订单明细中。

> 提示：每个商品在同一个订单中只能添加一次。如果尝试重复添加，系统会提示"该商品已在订单中"。

### 修改数量和单价

- **数量**：在订单明细的「数量」列中直接输入或调整数值，最小为 1。
- **成交单价**：在「成交单价」列中输入实际成交价格，默认使用商品销售价。
- **小计**：系统自动计算（数量 x 成交单价）。

### 确认销售

1. 确认订单明细无误后，点击底部的「确认销售」按钮。
2. 系统自动执行以下操作：
   - 创建订单（状态为"已确认"）
   - 扣减对应商品的库存
   - 如果选择了收款方式，自动创建收款记录
3. 页面跳转到订单详情页。

> 重要：确认销售后库存会立即扣减。如果库存不足，系统会提示错误。

### 填写备注

在订单底部的「备注」文本框中可输入补充说明（选填，最多 500 个字符）。

## 7.4 订单操作

### 确认订单

如果订单状态为"草稿"，可以点击「确认订单」按钮：

1. 系统弹出确认对话框："确认订单将扣减库存，确定？"
2. 点击确定后，订单状态变为"已确认"，对应商品库存被扣减。

> 注意：确认订单是不可逆操作，确认后库存立即扣减。

### 取消订单

如果订单状态为"草稿""已确认"或"部分收款"，可以点击「取消订单」按钮：

1. 系统弹出确认对话框："确定取消该订单？"
2. 点击确定后，订单状态变为"已取消"。
3. 如果订单之前已扣减库存（即已确认状态），系统会自动恢复库存。

> 说明：取消订单后，已扣减的库存会自动归还。

### 登记收款

如果订单状态为"已确认"或"部分收款"，可以登记收款：

1. 点击「登记收款」按钮。
2. 系统弹出收款登记对话框。
3. 查看剩余应收金额。
4. 输入收款金额（不能超过剩余应收金额，最小 0.01 元）。
5. 选择收款方式（现金/转账/微信/支付宝/其他）。
6. 可选填备注。
7. 点击「确认收款」完成登记。

> 说明：当累计收款金额等于订单金额时，订单状态自动变为"已完成"。

### 冲正收款

如果需要撤销某笔收款记录：

1. 在订单详情的收款记录区域，找到需要冲正的记录。
2. 点击该行的「冲正」按钮。
3. 系统弹出确认对话框："确定冲正该笔收款？"
4. 点击确定后，该笔收款被标记为"已冲正"状态。

> 注意：冲正操作会撤销该笔收款记录，但不改变订单状态。如需恢复，需重新登记收款。

### 编辑订单

如果订单状态为"草稿"，可以点击「编辑」按钮修改订单内容。

1. 点击右上角的「新建角色」按钮。

> 说明：编辑角色的权限后，拥有该角色的全部用户立即生效，无需重新登录。

## 12.5 删除角色

1. 在角色列表中找到要删除的角色，点击「删除」按钮。
2. 系统弹出确认对话框。
3. 点击确定完成删除。

> 注意：如果角色已经关联了用户（关联用户数 > 0），则删除按钮为灰色不可点击状态。必须先将该角色的全部用户调整为其他角色后才能删除。

# 第十三章 CSV 导入导出指南

本章详细说明 CSV 文件的导入导出操作。

## 13.1 导入商品

### 操作步骤

1. 在商品列表页点击右上角的「导入」按钮。
2. 系统弹出文件选择窗口。
3. 选择准备好的 CSV 文件。
4. 系统自动上传并处理，完成后显示导入结果。

### 文件要求

| 项目 | 限制 |
|------|------|
| 文件格式 | CSV（.csv 后缀） |
| 文件大小 | 最大 10MB |
| 行数限制 | 最多 1000 行（不含表头） |
| 编码格式 | UTF-8 |

### CSV 模板格式

表头支持中文或英文名称。以下是中文表头的模板：

```csv
商品名称,销售价,成本价,SKU,库存数量
纯棉T恤,99.00,45.00,SKU-TSHIRT-001,100
牛仔裤,199.00,80.00,,50
运动鞋,299.00,120.00,SKU-SHOE-001,30
```

### 字段说明

| 列名（中文） | 列名（英文） | 是否必填 | 说明 |
|--------------|--------------|----------|------|
| 商品名称 | name | 必填 | 商品名称 |
| 销售价 | sale_price | 选填 | 销售价格，如 99.00 |
| 成本价 | cost_price | 选填 | 成本价格，如 45.00 |
| SKU | sku | 选填 | 商品编码，留空则自动生成 |
| 库存数量 | stock_quantity | 选填 | 初始库存，默认为 0 |

> 提示：可以用 Excel 或 WPS 编辑 CSV 文件。编辑完成后，请确保保存时选择 UTF-8 编码，否则中文可能出现乱码。

### 在 Excel 中创建 CSV 文件

1. 打开 Excel，新建空白工作簿。
2. 在第一行输入表头：`商品名称,销售价,成本价,SKU,库存数量`（每个字段占一列）。
3. 从第二行开始逐行填写商品数据。
4. 点击「文件」->「另存为」，选择文件类型为「CSV (逗号分隔)(*.csv)」。
5. 保存文件。

> 警告：Excel 保存 CSV 时可能使用 GBK 编码而非 UTF-8。如果导入后中文乱码，请用记事本打开 CSV 文件，另存时将编码改为 UTF-8。

## 13.2 导入客户

### 操作步骤

1. 在客户列表页点击右上角的「导入」按钮。
2. 系统弹出文件选择窗口。
3. 选择准备好的 CSV 文件。
4. 系统自动上传并处理。

### 文件要求

与商品导入相同（CSV 格式，最大 10MB，最多 1000 行，UTF-8 编码）。

### CSV 模板格式

```csv
客户名称,电话,联系人,邮箱,来源,等级,备注
张三公司,13800138001,张三,zhangsan@example.com,referral,vip,老客户
李四,13900139001,,,,normal,
王五科技,13700137001,王五,wangwu@example.com,online,important,新签约客户
```

### 字段说明

| 列名（中文） | 列名（英文） | 是否必填 | 说明 |
|--------------|--------------|----------|------|
| 客户名称 | name | 必填 | 客户名称 |
| 电话 | phone | 选填 | 手机号，需唯一 |
| 联系人 | contact_name | 选填 | 联系人姓名 |
| 邮箱 | email | 选填 | 邮箱地址 |
| 来源 | source | 选填 | referral/online/offline/ad/other |
| 等级 | level | 选填 | vip/important/normal/potential |
| 备注 | remark | 选填 | 补充说明 |

### 来源和等级的可选值

**来源（source）**：

| 值 | 含义 |
|----|------|
| referral | 转介绍 |
| online | 线上 |
| offline | 线下 |
| ad | 广告 |
| other | 其他 |

**等级（level）**：

| 值 | 含义 |
|----|------|
| vip | VIP |
| important | 重要 |
| normal | 普通 |
| potential | 潜在 |

> 注意：CSV 中的来源和等级字段请使用英文值（如 `referral`、`vip`），不要使用中文。

## 13.3 导出数据

各模块的导出操作方法相同：

2. 设置好搜索和筛选条件（导出范围受当前条件影响）。
3. 点击「导出」按钮。
4. 浏览器自动下载 CSV 文件。

### 导出范围说明

| 导出操作 | 导出范围 |
|----------|----------|
| 商品导出 | 受搜索关键词和状态筛选影响 |
| 客户导出 | 受搜索关键词和来源筛选影响 |
| 订单导出 | 受搜索关键词和状态筛选影响 |

> 提示：如果要导出全部数据，请先清空所有搜索和筛选条件。

# 第十四章 常见问题（FAQ）

## 登录相关

**Q: 忘记密码怎么办？**

A: 请联系系统管理员重置密码。管理员可以在「用户管理」中删除您的账号并重新创建，或直接创建一个新的密码。

**Q: 登录时提示"账户已被锁定"，怎么办？**

A: 连续 5 次输入错误密码后，账户会被锁定 15 分钟。请等待 15 分钟后再尝试登录，或联系管理员处理。

**Q: 登录后一段时间页面自动跳转到登录页面，怎么办？**

A: 这是因为您的登录会话已过期。系统令牌有效期为 30 分钟，如果在 30 分钟内没有任何操作，令牌会失效。请重新登录即可。

## 商品相关

**Q: 创建商品时 SKU 应该怎么填？**

A: 建议留空，系统会自动生成格式为 `SPU-YYYYMMDD-NNNN` 的唯一编码。如果您有自有的编码体系，也可以手动输入。

**Q: 修改商品价格后，之前的订单价格会变吗？**

A: 不会。订单明细中保存的是下单时的价格快照，修改商品价格只影响后续新建的订单。

**Q: 商品停用后还能恢复吗？**

A: 可以。在商品列表中筛选"停用"状态的商品，进入编辑页面，将状态改为"上架"即可恢复。

**Q: 上传图片失败怎么办？**

A: 请检查以下几点：
- 图片格式是否为 JPG、PNG 或 WebP
- 图片大小是否超过 5MB
- 网络连接是否正常

## 订单相关

**Q: 创建订单时看不到某个商品？**

A: 订单创建页面只显示"上架"状态的活跃商品。如果商品被停用或下架，将不会出现在选择列表中。

**Q: 订单创建后可以修改吗？**

A: 只有"草稿"状态的订单可以编辑。已确认、已取消、已完成等状态的订单不能修改。

**Q: 取消订单后库存会恢复吗？**

A: 会的。取消已确认的订单时，系统会自动归还已扣减的库存，并在库存流水中生成"取消退货"记录。

**Q: 订单的收款金额大于订单金额怎么办？**

A: 系统限制收款金额不能超过剩余应收金额。如果需要多收，请修改订单金额后再登记收款。

**Q: 冲正收款后钱会自动退回吗？**

A: 冲正操作只是在系统中标记该笔收款为"已冲正"，实际的退款操作需要在线下完成。

## 客户相关

**Q: 添加客户时提示电话号码已存在？**

A: 系统要求每个电话号码唯一。请确认该电话号码是否已被其他客户使用。如果是同一客户重复录入，请使用编辑功能修改已有客户信息。

**Q: 客户可以没有电话号码吗？**

A: 可以。电话号码是选填字段，但一旦填写就必须唯一。

## 导入导出相关

**Q: 导入 CSV 时提示编码错误或中文乱码怎么办？**

A: 请确保 CSV 文件使用 UTF-8 编码保存。在 Windows 系统中，Excel 默认使用 GBK 编码保存 CSV，建议用记事本打开后另存为 UTF-8 编码。

**Q: 导入时最多能导入多少条数据？**

A: 单次导入最多支持 1000 行数据（不含表头），文件大小不超过 10MB。如果数据量超过限制，请分批导入。

**Q: CSV 表头用中文还是英文？**

A: 系统同时支持中文和英文表头。例如"商品名称"和"name"都可以识别。

**Q: 导出文件用什么软件打开？**

A: 导出的 CSV 文件可以用 Excel、WPS、Google Sheets 或任何文本编辑器打开。如果用 Excel 打开出现乱码，请先在记事本中确认编码是否正确。

## 权限相关

**Q: 为什么我看不到某些菜单或按钮？**

A: 菜单和按钮的显示受角色权限控制。如果您缺少某项功能的访问权限，请联系系统管理员为您分配相应的角色或权限。

**Q: 为什么看不到成本价和毛利信息？**

A: 成本价、利润和毛利率信息需要"查看成本价"权限才能显示。请联系管理员确认您的角色是否包含该权限。

## 系统相关

**Q: 系统支持哪些浏览器？**

A: 系统支持所有现代浏览器，推荐使用 Chrome、Edge、Firefox 或 Safari 的最新版本。不建议使用 Internet Explorer。

**Q: 数据多久更新一次？**

A: 所有数据均为实时更新。首页仪表盘和报表中心的数据在页面加载时自动获取最新数据，刷新页面即可更新。

**Q: 如何备份数据？**

A: 数据备份由系统管理员负责。如有需要，请联系管理员处理。
